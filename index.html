#表格

表格

表格是网页制作中使用的最多的工具之一。在制作网页时，使用表格可以更清晰的排列数据，但在实际制作过程中，表格更多的用于网页的布局定位。很多网页都是以表格布局的，这是因为表格在文本和图像的位置控制方面都有很强的功能。

表格由行、列和单元格三部分组成，一般通过3个标记来创建，分别是表格标记table、行标记br和单元格标记td。表格的各种属性都要放在table元素之间。

创建表格

表格有许多的属性来供我们选择，合理的使用表格的各种属性可以更好的设计我们想要的表格效果。

caption表格标题

我们可以使用caption来设置标题单元格，表格的标题单元格一般位于整个表格的第1行，一个table表格只能含有一个表格标题。

表头th

表格的表头th是td单元格的一种变体，实质上仍然是一种单元格。它一般位于第一行和第一列，用来标明这一行或列的内容类别，一般情况下，浏览器会以粗体和居中的样式显示th元素中的内容。

表单

表单主要用来收集客户端提供的相关信息，使网页具有交互功能。在制作网页的过程中，常常需要使用表单，如会员注册、网上调查和搜索等。访问者可以使用如文本域、列表框、复选框以及单选按钮之类的对表单对象输入信息，然后单击某个按钮提交这些信息。表单的标记是form，在form标记中可以设置表单的基本属性，包括表单的名称、处理程序和传送方法等。一般情况下，表单的处理程序action和传送方法method是必不可少的参数。表单的处理程序是表单要提交的地址，也就是表单中收集到的资料要传递的程序地址。这一地址是绝对地址，也可以是相对地址，还可以是一些其他形式的地址。

#图像

图像

图像是网页中不可或缺的元素，合理的使用图像可以为网页增色不少。网页中图像的格式有三种，分别是：GIF、JPEG和PNG。目前支持的最好的是GIF和JPEG，大多数浏览器都可以查看它们。由于PNG文件居右较大的灵活性并且文件较小，所以他对于几乎任何类型的网页图形都是最适合的，但是早期的浏览器只能部分支持PNG图像的显示，所以建议使用GIF或者JPEG格式，以满足更多人的需求。

GIF格式是图像交换格式，文件最多可以使用256种颜色，最大的优点就是可以制作动态图像，可以将数张静态文件作为动画帧串联起来，转换成一个动画文件。另外一个优点就是可以将图像以交错的方式在网页中呈现，交错显示就是当图像尚未下载完成时，浏览器会先以马赛克的形式将图像慢慢显示，让浏览者可以大致猜出下载图形的雏形。

JPEG是一种图像压缩格式，是用于摄影或者连续色调图像的高级格式，这是因为JPEG文件可以包含数百万中颜色。它专门用于不含大色块的图像，有一定的失真度，但是在正常的损失下分辨不出JPEG和GIF图像的区别，JPEG文件大小只有GIF文件的1/4，不支持透明图和动态图，但能够保留全真的色调板格式。如果图像要全彩模式才能表现效果的话，JPEG就是最佳的选择。

PNG图像格式是一种非破坏性的网页图像文件格式，它提供了将图像以最小的方式压缩却不造成图像失真的技术，他不仅具备了GIF图像格式的大部分优点，而且还支持48位的色彩，更快的交错显示，跨平台的图像亮度控制，更多层的透明度设置。

###超链接

每一个网页都有一个唯一的地址，称为统一资源定位符（URL），然而，在网页中创建内部链接时，一般不会指定链接文档的完整URL，而是指定一个相对当前文档或者站点根文件夹的相对路径。这其中又分为绝对路径和相对路径。它们各有优缺点；

绝对路径

绝对路径是包括完整服务器规范在内的完全路径，不管源文件在什么位置都可以准确的找到，除非是目标文档的位置发生变化，否则链接不会失败。

优点：它同链接的源端点无关，只要网站的地址不变，无论文档在站点中如何移动，都可以正确实现跳转，如果希望链接同站点上的其他内容，就必须使用绝对路径。
缺点：这种方式的链接不利于测试，必须在Internet服务器端对链接进行测试，采用绝对路径也不利于站点的移植。

相对路径

相对路径可以表述源端点同目标端点之间的相互位置，它同源端点的位置密切相关。

如果连接中源端点和目标端点位于同一个目录下，则在链接路径中只需要指明目标端点的文档名称就可以。

如果链接中源端点和目标端点不在同一个目录下，则需要将目录的相对关系表示出来就可以了。

如果链接指向的文档位于当前目录的子级目录中，可以直接输入目录名称和子目录名称。

如果链接指向的文档没有位于当前目录的子级目录中，可以利用…符号来表示当前位置的父级目录，多个…符号可以表示更高级的父级目录。

优点： 如果站点的结构和文档的位置不变，那么链接就不会出错，可以将整个网站移植到另一个地址的网站中，而且不需要修改文档中的链接路径。

内部链接

内部链接是指链接的对象是在同一个网站中的资源，与自身网站页面有关的链接被称为内部链接。

#####理解和使用音频会话

音频会话简单的理解就是可以在我们使用音乐播放的过程中，收到电话通知，这时候就需要有一个管理的处理来将音乐播放停止，开始播放来电铃声。

所有的iOS应用程序都具有音频会话，无论其是否使用。AVFoundation定义了七种分类来描述应用程序锁使用的音频行为

  • Ambient 游戏、效率应用程序
  • Solo Ambient 游戏、效率应用程序
  • Playback 音频和视频播放器
  • Record 录音机、音频捕捉
  • Play and Record VoIP、语音聊天
  • Audio Processing 离线会话和处理
  • Multi-Route 使用外部硬件的高级A/V应用程序

其中，来实现播放音频的方法很简单，将本地要播放的文件放入我们的项目中，新建一个播放器就可以轻松实现音频的播放的功能了。

我们也可以自己修改音乐播放器的音量，这个是独立于系统的音量，还有一个需要我们注意的就是当音频被打断的时候的处理，我们需要监听线路的变化，并作出相应的处理。相关的方法是AVAudioSessionRoute…相关的类，这里大家可以自行查询。

iOS 中的内存管理

iOS中的内存管理

内存是我们平常生活中用来存储数据的存在，软件在运行的时候会分配和使用设备的内存资源，因此需要我们在软件开发的过程中，进行合理有效的内存管理，以保证快速高效的内存分配，提升软件的优越性。

首先我们来讨论一下内存中的分区：

1.内存中分为五大区域

在这里我们只做简单的介绍

栈区：存储局部变量的区域，当其作用域执行完毕之后，就会被系统收回，是系统自动管理的。

堆区： 存储OC的对象，手动申请的字节空间，需要自己释放

BSS段（Block Started by Symbol）：未初始化的全局变量和静态变量，一旦初始化就会从BSS段中回收掉，转存到数据段中

数据段： 存储已经初始化的全局变量和静态变量，以及常量数据，直到程序结束时才会被立即收回。数据段属于静态内存分配。

代码段： 通常是指用来存放程序执行代码的一块内存区域，这部分区域的大小在程序运行前就已经确定了，并且内存区域通常属于只读。在代码段中，也有可能包含一些只读的常量数据。

除了堆区，其他区域中存储的数据，都是由系统自动释放的。堆区中存放的OC的对象，是不会自动释放的，如果不主动释放，那么将在程序结束的时候释放。

内存的管理原则

就是我们常说的引用计数的问题，OC的对象都继承于NSObject，该对象有一个属性，叫做retainCount,也就是我们说的引用计数器，类型为unsigned long，占据八个字节，每一个对象负责维护对象所引用的计数值，当一个新的引用指向对象，那么这个对象的引用计数会增加1，当我们新创建一个对象的时候这个对象的引用计数器的值默认为1，当这个对象被少一次引用的时候这个对象的引用计数值减1，当这个对象的引用计数值为0的时候就说明了这个对象没有被使用，这个时候系统会自动回收掉此对象。

内存管理的方式：

1.MRC: Manual Reference Counting 手动引用计数器，需要我们手动管理对象引用计数的值。

2.ARC： Automatic Reference Counting 自动引用计数，系统自动的改变对象引用计数的值（iOS5之后），ARC是基于MRC的

这里我们重点讨论一下ARC：

ARC是automatic reference counting 自动引用计数，在程序编译时自动加入retain/release。 在对象被创建时retaincount + 1,在对象被release时count -1 ，当count=0时，对象销毁。程序中加入autoreleasepool对象会由系统自动加上autorelease方法，如果该对象引用计数为0，则销毁。ARC是为了解决MRC手动管理内存存在的一些问题而诞生的。

MRC下内存管理的缺点：

  • 释放一个堆内存时，首先要确定指向这个堆空间的指针都被release了。（要避免提前释放）
  • 释放指针指向的堆空间，首先要确定哪些指向同一个堆，这些指针只能释放一次。（避免释放多次，造成内存泄漏）
  • 模块化操作时，对象可能被多个模块创建和使用，不能确定最后由谁来进行释放。
  • 多线程操作时，不确定那个线程最后使用完毕。

但是ARC下也可能会出现内存泄漏的情况，比如如下的情况：

  • block的循环引用，可以将引用的一方变成weak，来避免循环引用。
  • NSTimer 在使用NSTimer addTarget时，为了防止target被释放而导致的程序异常，timer会持有target，这个也是一处内存泄漏的隐患。
  • 非OBJC对象 ARC是自动检测objc对象的，如果不是objc对象就无能为力了，C语言使用malloc开辟，free释放 c++使用new开辟，delete释放，但是在ARC下，不会添加非objc对象释放语句，如果没有去释放，也会造成内存泄漏。

每一个工程中都有一个main.m文件

int main(int argc, char * argv[]) {

@autoreleasepool {

return UIApplicationMain(argc, argv, nil, NSStringFromClass([AppDelegate class]));

}

}

autoreleasepool是用来做autorelease变量释放的，autorealease不会马上释放，当他到了最近的pool release时会检查retain count是不是为0，为0就会释放。

自动释放池的底层实现：

自动释放池以栈的形式实现： 当你创建一个新的自动释放池时，他将被添加到栈顶，当一个对象收到发送autorelease消息时，他被添加到当前线程的处于栈顶的自动释放池中，当自动释放池被回收时，他们从栈中被删除，并且会给池子里所有的对象都会做一次release操作。
