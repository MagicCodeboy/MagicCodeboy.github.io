HTML5中的语法以及标记变化和结构

Posted on 2020-04-05 | In HTML5 | 浏览 次
Words count in article: 2,413 字 | Reading time ≈ 9 分钟

HTML5基础语法

HTML5是2010年正式推出的，在新的HTML5语法规则中，部分JavaScript代码将被HTML5的新属性所替代，部分Div布局代码也将被HTML5变为更加语义化的结构标签，这使得网站前端的代码变的更加精炼、简介和清晰，代码所表达的意思也更加一目了然。HTML语法是在SGML语言的基础上建立起来的，但是SGML语法非常复杂，要开发能够解析SGML语法的程序也很不容易，所以很多浏览器都不包含SGML的分析器。虽然HTML基本遵从SGML的语法，但是对于HTML的执行在各个浏览器之间并没有统一的标准。

HTML5提供了高效的数据管理、绘制、视频和音频工具，促进了网页和移动设备的跨浏览器应用的开发。HTML5拥有强大的灵活性，支持开发非常精彩的交互式网站。还引入了新的标签和增强性的功能，其中包括了表单控制、API、多媒体、数据库支持和显著提升的处理速度等。

HTML5取消了HTML4.01中的一部分被css取代的标记，提供了新的元素和属性，语言更加精简，解析的规则更加详细。部分元素对于搜索引擎更加友好，对于小屏幕的设置和视障人士也更有帮助。HTML5采用了最新的表单输入对象，还引入了微数据。

HTML5语法的三个要点

1.可以省略标签的元素

不允许写结束符的标签：

area、base、br、col、command、embed、hr、img、input、keygen、link、meta、param、source、track、wbr

可以省略结束标签：

li、dt、dd、p、rt、rp、optgroup、option、colgroup、thead、tbody、tfoot、tr、td、th

可以省略整个标签：

HTML、head、body、colgroup、tbody 虽然这些元素都可以省略，但实际上却是隐形存在的。例如body标签可以省略，但在DOM树上他是存在的，可以使用document.body来访问到。

2.取得boolean值的属性

取得Boolean的属性，例如disabled和readonly等，通过默认属性来表达值为true，在属性值为true时，可以将属性值设置为属性名称本身，也可以将值设置为空字符串。

3.省略属性的引用符

在HTML4中设置属性值时，可以使用双引号或者单引号来引用。

在HTML5中，只要属性不包含”空格“、”<“、”>”、”‘“、”””、“、”、“=”等字符，都可以省略属性的引用符。

HTML5中新增的元素和废除的元素

HTML5增加了新的结构元素来表达最常用的结构。

结构元素 作用用法介绍
section: 可以用来表达书本的一部分或者一章，或者一章内的一节
header 页面主体的头部，并非head元素
footer 页面的底部（页脚），可以使一封邮件签名的所在
nav 到其他页面的链接集合
article 博文、杂志、文章汇编等中的一篇文章
块级元素 作用用法介绍
aside 定义页面内容之外的内容，比如侧边栏。可以表达注记、侧栏、摘要、插入的引用等作为补充主体的内容。
figure 定义媒介内容的分组，以及他们的标题。表示一段独立的流内容，一般表示文档主题流内容中的一个独立单元。
figcaption 媒介内容的标题说明。使用figcaption元素可以为figure元素组添加标题。
dialog 定义对话（会话）。使用dialog元素用来表达人们之间的对话。在HTML5元素中，dt元素用于表示说话者，而dd元素用来表示说话的内容。
行内元素 作用用法介绍
mark 定义有记号的文本
time 定义日期/时间
meter 定义预定义范围内的度量,可以用来展示投票结果等 meter value=“30” optimum=“1” high=“0.9” low=”1” max=”1” min=”0”
progress 定义运行中的进度
嵌入多媒体元素与交互性元素 作用用法介绍
video 定义视频
audio 定义音频
embed 用来插入各种多媒体，格式可以使MIDI，WAV，AIFF，AU，MP3等
input元素 作用用法介绍
url 专门用来输入url地址的文本框,如果文本框中的内容不是url地址格式，则不允许提交
email 设置这个值会在提交表单的时候自动验证输入框中的内容是否为email格式，如果不是，无法提交
data 以日历的形式来方便用户输入
time 用来输入时间的本文框，并且会对输入时间的有效性进行检查
datetime 专门用来输入本地日期和时间的文本框，也会对输入的数据进行检查
废除的元素 代替属性
acronym abbr
dir ul
isindex 使用form元素与input元素相结合的方式替代
listing pre
xmp code
nextid GUIDS
plaintext “text/plian”MIME类型替代

更多的被废除的属性可以查看官方文档，这里就不一一列举了

新增的属性

input表单新增的属性：

新增属性 属性定义 设置
input（type=text）、select、textarea与button 新增autofocus属性 指定属性的方式让元素在画面打开时自动获得焦点
input（type=text）、textarea指定placeholder属性 对用户的输入进行提示，提示用户可以输入的内容
input、output、select、textarea、button与fieldset指定form属性 声明属于哪个表单，然后将其放置在页面的任何位置，而不是表单之内
input（type=text）、textarea指定required属性 表示用户提交时进行检查，检查该元素内必定要有输入内容
input 标签增加新的属性 autocomplete、min、max、multiple、pattern、step datalist元素与autocomplete属性配合使用，multiple属性允许上传时一次上传多个文件，pattern属性用于验证输入字段的模式、其实就是正则表达式，step属性规定输入字段的合法数字间隔 如：（step=“3”，则合法数字范围是-3、0、3、6…），step属性可以与min以及max属性配合使用，创建合法值的范围
input、button元素增加formaction、formenctype、formmethod、formnovalidate与formtarget属性 用户重载form元素的action、enctype、method、novalidate与target属性，
fieldset元素增加disabled属性 可以把他的子元素设置为disabled状态
input、button、form增加novalidate属性 可以取消提交时进行的有关检查，表单可以被无条件的提交

链接相关的属性：

新增属性 属性定义 设置
a、area增加media属性 规定目标URL是以何种类型的媒介、设备进行优化的，改属性可以接受多个值，只有在href属性存在时使用
area增加hreflang和rel属性 hreflang属性规定在被链接文档中的文本的语言，只有设置了href属性时，才使用该属性。rel属性规定当前文档与被链接文档/资源之间的关系，只有href使用时，才能使用rel属性
link增加size属性 用来规定被连接资源的尺寸，只有被链接资源是图标时（rel=”icon”）,才能使用该属性，该属性可以接受多个值，值由空格分隔
base元素增加target属性 保持与a元素的一致性

其他的新增的属性

新增属性 属性定义 设置
ol增加reversed属性 指定列表倒序展示
meta增加charset属性 规定在外部脚本文件中使用的字符编码
menu增加type和label属性 label为菜单定义一个可见的标注，type属性让菜单可以以上下文菜单、工具条与列表菜单三种形式出现
style增加scoped属性 允许我们为文档的指定部分定义样式，而不是整个文档，如果使用scoped属性，那么所规定的样式只能应用到style元素的父元素及其子元素
script增加async属性 定义脚本是否异步执行，仅适用于外部脚本（使用src标记时）
HTML元素增加manifest 开发离线Web应用时它与API结合使用
iframe增加sandbox、seamless、srcdoc属性 用来提高页面安全性，防止不信任的网页执行某些操作

新增的主体结构元素

结构元素 介绍
article 可以包含一个独立的内容项，每一个article元素都允许有自己的标题与注脚等从属元素，并允许对自己的从属元素单独使用样式
section 用于对网站或APP中页面上的内容进行分块，一个section元素通常由内容和标题组成。
nav 在HTML中用于包裹一个导航链接组，在一个页面中可以同时存在多个nav
aside 用来表示当前页面或者文章的附属信息部分，它可以包含与当前页面或者主要内容相关的引用，有别于主要内容的部分

新增的主体结构元素

元素 介绍
header 是一种具有引导和导航作用的结构元素，通常用来放置整个页面或者页面内的一个内容区块的标题，也可以包含表格、表单或者相关的logo图片
hgroup 位于正文开头，可以在这些元素中添加h1标签，用于显示标题，hgroup元素可以将标题及其子标题进行分组的元素。
footer 通常包括其相关区块的脚注信息
address 通常位于文档的末尾，用于在文档中呈现联系信息，包括文档创建者的名字、站点链接、电子邮箱、真实地址、电话号码等
